OpenAI Cached Pricing: Cut Input Costs by 50%

Prompt caching is the most underutilized cost-saving feature in the AI ecosystem. When enabled, GPT-4o's input price drops from $2.50 to $1.25 per 1M tokens — a 50% reduction with zero quality loss.

How Prompt Caching Works

When you send repeated requests with the same system prompt or long prefix, OpenAI automatically caches those tokens. Subsequent requests that share the same prefix get billed at 50% of the standard input rate.

The 1,024-Token Threshold

Only prefixes of 1,024 tokens or longer are eligible for caching. The cached portion must be identical across requests. Even one token difference invalidates the cache for that request.

Edge Cases & Gotchas

⚠️ Things That Break Caching

  1. Varying system prompt: Even one word difference → cache miss. Solution: put all dynamic content in user message.
  2. Adding timestamps: `You are a bot. Current time: {now()}` → never cached. Solution: pass time in user message.
  3. Different tool definitions: If you vary function schemas per request → cache miss. Keep tools static.
  4. Short prompts: <1,024 tokens → never cached. Only worth it for substantial system prompts or RAG contexts.
  5. Cache eviction: OpenAI doesn't guarantee cache retention. Popular prefixes are retained longer. No SLA.
